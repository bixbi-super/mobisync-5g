What's New in v4.0.5

NEW

Community Edition introduced

The same binary now runs as Community Edition when no license.lic file is present. No separate download, no registration — plug in, run, see live RRC OTA in Wireshark immediately. Full feature access unlocks automatically when a valid license.lic is placed alongside the executable.

NEW

Live RRC OTA decode — Community Edition

Community Edition provides full RRC OTA decode: MIB, SIB1/SIBs, RRC Setup, RRC Reconfiguration, Security Mode, UE Capability, Paging and more — all streamed live into Wireshark via standard PCAP.

IMPROVED

Edition detection at startup

The title bar now clearly identifies the running edition — MobiSync-SA-5G Logger: Community Edition or MobiSync-SA-5G Logger: Professional Edition. Log categories unavailable in Community are greyed out in the GUI rather than hidden.

IMPROVED

setup.sh — single-step dependency installer

Installs ADB via apt, configures udev rules for phone detection, and verifies the environment in one step. Run once, then launch the binary directly on every subsequent session.

Installation — Ubuntu

Two files are included in this release. Download both into the same folder before proceeding.

FilePurpose
mobisync_v4p0p5-5g-logger_community_ednMain application binary (Community + Professional)
setup.shOne-time dependency installer and environment setup
1

Enable Developer Options on the phone

Go to Settings → About Phone and tap Build Number seven times. Then go to Settings → Developer Options and enable USB Debugging.

2

Connect the phone via USB

Use a USB data cable (not charge-only). When prompted on the phone, select File Transfer (MTP) mode and tap Allow on the USB Debugging authorisation dialog.

3

Run setup.sh (once only)

Open a terminal in the folder containing both files and run:

# Make executable and run chmod +x setup.sh ./setup.sh

This installs ADB via apt, adds udev rules so the device is detected without root, and verifies your environment. A confirmation message is shown on success.

ℹ
Run setup.sh only once per installation. All subsequent launches go straight to step 4.
4

Launch Mobisync

Make the binary executable and launch it:

chmod +x mobisync_v4p0p5-5g-logger_community_edn sudo ./mobisync_v4p0p5-5g-logger_community_edn

The title bar shows MobiSync-SA-5G Logger: Community Edition confirming no license was detected.

5

Select log types and start logging

In the RRC & NAS Logs panel, check Enable NRSG RRC OTA in Wireshark. MAC, L1, and NAS options are greyed out in Community Edition.

Click Start Logging. Wireshark opens automatically and begins filling with live decoded RRC messages from the UE.

6

Stop logging

Click Stop Logging in the Mobisync GUI. The Wireshark capture can be saved as a .pcapng file for offline analysis on any machine with Wireshark 4.4.2+ installed.

ℹ
Upgrading to Professional Edition: Obtain a license.lic file from info@makemytechnology.com, place it in the same folder as the binary, and relaunch. No reinstallation needed — the same binary activates the full feature set automatically.

Known Issues

Samsung / Xiaomi DIAG port locked: Some firmware variants lock the Qualcomm DIAG port. Root access is required on these devices to expose the port to ADB.
Wireshark version below 4.4.2: Older Wireshark builds are missing some 3GPP NR dissectors. Upgrade to 4.4.2 or later for complete RRC decode.
ADB device not detected after setup.sh: Unplug and replug the USB cable, or run adb kill-server && adb start-server to reset the ADB daemon.
Single device only: Multi-device simultaneous capture is on the roadmap. Only one phone can be logged per session in v4.0.5.
